Quartzine

// USB interception & sandboxed analysis for malware labs

What it does

Quartzine sits between your hardware and your OS. The moment a USB drive is plugged in, it intercepts the event before the kernel mounts anything — and asks what you want to do with it.

Mount it normally, or hand it to a disposable VM that will open it, explore it, and let you watch exactly what happens.

Flow

USB plugged in
│
├─ udev intercepts (before udisks2 mounts)
│
├─ Mount normally? ──────────────────── normal OS behavior
│
└─ Analyze?
│
├─ Spin up clean VM snapshot
├─ USB passthrough → VM
├─ VM opens & navigates the drive
├─ eBPF + network tap observe everything
├─ Report generated
└─ VM destroyed. Nothing persists.

Key properties

⚡
Hardware-triggered

Intercepts at plug-in, not at file submission. No manual steps.

🎭
Deceptive environment

VM mimics a real user machine. Masked hypervisor artifacts, fake activity history.

👁
Host-side observation

eBPF probes from the host. Invisible to anything running inside the VM.

🔌
Isolated network

Simulated internet via INetSim. The VM never touches real infrastructure.

🗑
Ephemeral by design

Every analysis starts from a clean snapshot. Destroyed on completion.

📄
Structured reports

Network calls, filesystem changes, processes — exported as JSON.

Stack

Detection udev rules pyudev
Orchestration Python libvirt-python QEMU/KVM
Observation eBPF bpftrace Zeek
Network sim INetSim tcpdump
VM snapshots LVM ZFS

Requirements

# Host OS
Linux with KVM support (x86_64)
 
# Dependencies
$ pip install pyudev libvirt-python
$ apt install qemu-kvm libvirt-daemon bpftrace zeek inetutils
⚠ Warning This tool is intended for controlled lab environments only. Never use on production machines or with drives from untrusted sources outside a properly isolated network.
